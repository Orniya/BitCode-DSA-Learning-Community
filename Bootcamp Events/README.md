# Bootcamp Events — Weekly Contests & Mock Interviews

Shared rhythm across **Beginner**, **Intermediate**, and **Advanced** cohorts. Each applicant is **registered at one level for the full bootcamp** — tracks run **in parallel**, not as a ladder (nobody is required to “graduate” into another track mid-summer).

Roadmap phases live in each track README. This file covers **contest day** and **peer mock interviews** only.

---

## 📅 Weekly Codeforces mashup

**One contest day per week** for the whole bootcamp. Everyone competes on the same schedule; problems are split by **division** so difficulty matches what that cohort has covered recently.

| Division | Who | Typical problem pool (rotates with curriculum) |
| --- | --- | --- |
| **Beginner** | Beginner cohort only | Implementation, arrays/strings, hashing, stacks/queues, simple recursion — no trees/graphs/DP |
| **Intermediate** | Intermediate cohort only | Two pointers, sliding window, binary search, linked lists, trees, graphs, backtracking, intro DP |
| **Advanced** | Advanced cohort only | Hard mediums, advanced graphs/DP, greedy, tries, DSU, segtree/BIT, strings, bits |

**Format:** Custom **Codeforces mashup** prepared by mentors (not a full CF rated round). Divisions are separate scoreboards or problem subsets in one contest package.

**Expectations:**

- Beginner: focus on finishing 1–2 problems cleanly; partial solutions count as learning
- Intermediate: aim for 2–3 problems; write complexity after the contest
- Advanced: treat as timed practice; upsolve anything unsolved within 48 hours

🔗 Contest hub: *(link to weekly mashup — added when each round is published)*

**Track leads:** Announce the week’s topic focus in the Telegram group before contest day so each division aligns with recent lectures.

## 🎤 Peer mock interviews ([Exponent Practice](https://www.tryexponent.com/practice))

We run mocks on **[Exponent Practice](https://www.tryexponent.com/practice)** (peer matching, shared code editor, structured feedback — same product family as the former Pramp). Each session is **reciprocal**: you and your partner each spend ~**30 minutes** as interviewee and ~**30 minutes** as interviewer (~**1 hour** total).

**Interview type on Exponent:** **Data Structures & Algorithms**

**How to sign up:**

1. Create a free account at [tryexponent.com/practice](https://www.tryexponent.com/practice).
2. Click **Schedule practice session** → choose **Data Structures & Algorithms**.
3. Pick date/time and a **difficulty** that matches your cohort (Intermediate: mostly medium; Advanced: medium/hard).
4. **Match with a peer** via Exponent’s matcher, **or** use **Practice with a Friend** to pair with someone from your BitCode cohort at the same level.
5. Join from the practice page up to **10 minutes before** start time (video + collaborative editor; **Python** and **C++** are supported).

Mentors will post **mock windows** in Telegram (suggested weekends/evenings) so many people from the same cohort book overlapping slots — easier to match with bootcamp peers.

**Beginner cohort:** No required Exponent mocks. Use weekly mashups, phase checkpoints, and informal peer review.

### Intermediate cohort — 2 Exponent sessions

| Mock | Suggested timing | What to practice |
| --- | --- | --- |
| **Intermediate Mock 1** | After **Phase 6** (intro DP) | Medium spanning patterns → graphs/intro DP; full loop: clarify → approach → code → test → complexity |
| **Intermediate Mock 2** | After **Phase 8** (timed practice) or late Phase 7 | Medium under light time pressure; exchange written peer feedback on Exponent after the session |

### Advanced cohort — 2 Exponent sessions

| Mock | Suggested timing | What to practice |
| --- | --- | --- |
| **Advanced Mock 1** | After **Phase 3** (advanced data structures) | Medium/hard; edge cases, correctness, complexity |
| **Advanced Mock 2** | After **Phase 6** (design & integration) | Medium/hard with follow-ups on trade-offs and optimizations |

**Prep (Intermediate & Advanced):**

- Use the same language as daily practice (**Python** default; **C++** fine if that’s your norm).
- Skim the assigned question beforehand if Exponent surfaces one you’ve seen — interviewers often get familiar problems.
- After each session: save **peer feedback** from Exponent and note 1–2 fixes in your error log. Mocks are **feedback**, not pass/fail gates.
- As **interviewer**, use a simple rubric: clarification, structure, correctness, complexity, communication.

**Note:** Exponent’s free tier includes monthly practice credits; [paid membership](https://www.tryexponent.com/practice) adds unlimited sessions and optional AI feedback. Plan your **two** bootcamp mocks within the free allowance if possible.
